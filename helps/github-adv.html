GitHub Features Recursos do GitHub

GitHub is a powerful platform for version control and collaboration. Here are some of the key features that make it a popular choice among developers: GitHub é uma plataforma poderosa para controle de versão e colaboração. Aqui estão alguns dos principais recursos que o tornam uma escolha popular entre os desenvolvedores:

How to delete a repository and reset VSCode Como excluir um repositório e redefinir o VSCode

To delete a repository you must go to the settings tab inside the repository on GitHub. Para excluir um repositório, você deve ir para a aba de configurações dentro do repositório no GitHub.

Once there, scroll to the very bottom of the page, you will see a section called "Danger Zone". Uma vez lá, role até o final da página e você verá uma seção chamada "Danger Zone".

Click on the "Delete this repository" button. Clique no botão "Delete this repository".

Once you have clicked on the button, you will be asked to confirm that you want to delete the repository. Depois de clicar no botão, você será solicitado a confirmar que deseja excluir o repositório.

To confirm that you want to delete the repository, you must type the name of the repository in the text box. Para confirmar que deseja excluir o repositório, você deve digitar o nome do repositório na caixa de texto.

Once you have typed the name of the repository, click on the "I understand the consequences, delete this repository" button. Depois de digitar o nome do repositório, clique no botão "I understand the consequences, delete this repository".

You will be asked to enter your GitHub password as the items in the Danger Zone are administration activities and you must authenticate to use them. Você será solicitado a inserir sua senha do GitHub, pois os itens na Danger Zone são atividades de administração e você deve se autenticar para usá-los.

Note that deleting a repository will delete all the issues, pull requests, and wiki pages associated with the repository. Observe que excluir um repositório excluirá todos os issues, pull requests e páginas de wiki associados ao repositório.

Also note that deleting a repository will not delete the local repository on your computer, so you will need to delete the local repository manually. Observe também que excluir um repositório não excluirá o repositório local no seu computador, portanto você precisará excluir o repositório local manualmente.

These steps unlinks your local folder with GitHub Estes passos desvinculam sua pasta local do GitHub

Open the root folder of the repository in VSCode. Abra a pasta raiz do repositório no VSCode.

Open a terminal in VSCode. On Windows you use Ctrl+` and on a MAC you use ⌃` or you can use the menu option VIEW - Terminal. Abra um terminal no VSCode. No Windows, use Ctrl+` e no MAC, use ⌃` ou pode usar a opção de menu VIEW - Terminal.

Once the terminal is open, type the following command: Depois que o terminal estiver aberto, digite o seguinte comando:

On Windows: No Windows: 

Remove-Item .git -Recurse -Force

On a Mac and Linux: Em um Mac e Linux: 

rm .git -rf

These commands removes the tracking folder that GIT uses when syncing with GitHub. This hidden folder contains all the configuration information and tracking the changes to know what to sync with GitHub. Esses comandos removem a pasta de rastreamento que o GIT usa ao sincronizar com o GitHub. Esta pasta oculta contém todas as informações de configuração e rastreamento das alterações para saber o que sincronizar com o GitHub.

Now exit VSCode and open your file manager / File explorer and navigate to the folder where your local repository is located. Agora saia do VSCode e abra seu gerenciador de arquivos / Explorador de arquivos e navegue até a pasta onde seu repositório local está localizado.

Now rename the folder to a new name. Agora renomeie a pasta para um novo nome.

How to rename a repository Como renomear um repositório

To rename a repository you must go to the settings tab inside the repository on GitHub. Para renomear um repositório, você deve ir para a aba de configurações dentro do repositório no GitHub.

Once there, you will see a section called "Repository name". Uma vez lá, você verá uma seção chamada "Repository name".

Click on the name of the repository and change it to the new name. Clique no nome do repositório e mude-o para o novo nome.

Once you have changed the name, click on the "Rename" button to save the changes. Depois de ter alterado o nome, clique no botão "Rename" para salvar as alterações.

Note that renaming a repository will change the URL of the repository, so you will need to update any links to the repository. Observe que renomear um repositório mudará o URL do repositório, portanto você precisará atualizar quaisquer links para o repositório.

Also note that renaming a repository will not change the name of the local repository on your computer, so you will need to update the remote URL in your local repository. Observe também que renomear um repositório não mudará o nome do repositório local no seu computador, portanto você precisará atualizar o URL remoto no seu repositório local.

To update the remote URL in your local repository, you can use the following command: Para atualizar o URL remoto no seu repositório local, você pode usar o seguinte comando: 

git remote set-url origin new-url

Where new-url is the new URL of the repository. Onde new-url é o novo URL do repositório.

You can find the new URL by going to the Code tab and then click on the <> Code button Você pode encontrar o novo URL indo para a aba Code e clicando no botão <> Code

Once you have clicked on the button, you will see a tab called Local, make sure it is selected. Then make sure HTTPS is selected and the URL will be shown. Depois de clicar no botão, você verá uma aba chamada Local, certifique-se de que está selecionada. Em seguida, certifique-se de que HTTPS está selecionado e o URL será mostrado.

Click on the "Copy" button to copy the URL to your clipboard. The copy button is the icon to the right of the URL. You can then just paste the URL into the terminal where you are typing your command. Clique no botão "Copy" para copiar o URL para a área de transferência. O botão de copiar é o ícone à direita do URL. Você pode então colar o URL no terminal onde está digitando seu comando.

For example, if you renamed your repository from wdd130 to wdd130-old, the new URL would be https://githubusername.github.io/wdd130-old. Por exemplo, se você renomeou seu repositório de wdd130 para wdd130-old, o novo URL seria https://githubusername.github.io/wdd130-old.

This is the full command you would type remember to use your GitHub Username Este é o comando completo que você digitaria, lembre-se de usar seu nome de usuário do GitHub 

git remote set-url origin https://githubusername.github.io/wdd130-old

To check the remote URL of your local repository, you can use the following command: Para verificar o URL remoto do seu repositório local, você pode usar o seguinte comando: 

git remote -v
How to change a repository visibility Como alterar a visibilidade de um repositório

To change a repository visibility you must go to the settings tab inside the repository on GitHub. Para alterar a visibilidade de um repositório, você deve ir para a aba de configurações dentro do repositório no GitHub.

Once there, you will see a section called "Danger Zone". Uma vez lá, você verá uma seção chamada "Danger Zone".

Click on the "Change repository visibility" button. Clique no botão "Change repository visibility".

Once you click on the button, you will see the option attached to the button to change to the opposite of what the current visibility is. If it is private, you will see "Change to Public". Depois de clicar no botão, você verá a opção anexada ao botão para mudar para o oposto da visibilidade atual. Se for privado, você verá "Change to Public".

You will then be prompted to verify you want to make the change and you may have to enter your password to verify you identity and make the change. GitHub will only ask for your password about every 2 hours from the last time you entered the password. Você será solicitado a verificar se deseja fazer a alteração e pode ter que inserir sua senha para verificar sua identidade e fazer a alteração. O GitHub só pedirá sua senha a cada 2 horas aproximadamente desde a última vez que você inseriu a senha.
