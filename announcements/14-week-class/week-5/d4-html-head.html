This is a post that explains why you need to move the loading of the Stylesheet to after the <title> tag. If you do not care just make sure that the loading of the stylesheet is in the correct place. A lot of the info in this announcement is something that you will only use outside of school work. If you are going to get the social media certificate, this will be good info to know to help with social media tracking.

 

A lot of you have done something like this

<!DOCTYPE html>
<html lang="en">
<head>
    <link rel="stylesheet" href="styles/style.css">
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Whitewater Rafting Vacations | Dry Oar Boating | Home</title>
</head>
<body>

 or you have done this

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <link rel="stylesheet" href="styles/style.css"> 
    <title>Whitewater Rafting Vacations | Dry Oar Boating | Home</title>
</head>
<body>

 

The correct way to format your <head> area is like this with the link to the stylesheet after the <title> tag. 

<!DOCTYPE html>
<html lang="en">
<head>
    
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Whitewater Rafting Vacations | Dry Oar Boating | Home</title>
    <link rel="stylesheet" href="styles/style.css">
</head>
<body>

 

As you learn more about what can and does go into the <head> area of an HTML file, you will notice that there is a lot of confusion in this. The thing you have to keep in mind is this.

Stylesheets must load after the the <title> and the meta data that defines the type of text on the page and before the JavaScript files which you will learn about in a few weeks. Some of the meta data tells the browser what type of document, language and encoding to use with what is being loaded. If the browser does not know this info then you can get some unexpected results. This is especially true when working with multiple languages around the world. For example Japanese and Chinese browsers default to UTF-16 & UTF-32 to display characters on the screen. This can lead to unexpected display issues.

 

This is additional Information . You can read about the default <meta> tags here: https://www.w3docs.com/learn-html/html-meta-tag.html

 

There are a lot of<meta> type tags that are placed in the <head> tag. You will learn most of these on the job as they are not taught in many classes, even online classes/tutorials as they are specific to the project you are working on.

Here is the definition of the <meta> tag:

The <meta> tag contains metadata, which specifies page description, keywords, author of the document, etc. The metadata is used by browsers, search engines, and other web services, and isn’t displayed on the web page. A web document can contain more than one <meta> tag, but generally, the physical appearance of the document is not affected by it. 

With this definition in mind, you can see why search engines look to this tag in the <head> area. CSS and JavaScript can override these things and for that reason they should always be loaded after all the other tags in the <head> area.

There are other tags that are used for Social Media linking. Such as OpenGraph by Facebook and the Twitter tags. You can read the details here:

https://www.kontentino.com/blog/social-media-meta-tags-a-complete-guide/

      <meta property="og:image" content="https://ia.media-imdb.com/images/rock.jpg" />

      <meta property="og:description" content="Learn about 13 features that set A hrefs apart from the competition." />

      <meta property="og:locale" content="en_US" />
