All Site Plan submission must be the GitHub pages URL.

For example this would be what I would submit: https://borcherds-ralph.github.io/wdd130/wwr/site-plan-rafting.html

The URL cannot and must not start with http://github.com/

There is no exception to this submission rule.

 

Some additional tips

** Be sure to have the correct GitHub user name and password ready so you can connect your computer to your GitHub account.

** Make sure that your repository is set to Public. If you accidentally set it to private, follow this tutorial to make it public (Scroll down to "Changing a repository's visibility":

https://docs.github.com/en/repositories/managing-your-repositorys-settings-and-features/managing-repository-settings/setting-repository-visibility

** Be sure to follow all of the instructions in step 05 - Setting up GitHub Pages. Note that most of us will have "master" as the branch name.

If the page says, "'Your site is ready to be published at ...' it worked!! You may need to be patient. It may take several minutes for your page to show up. Sometimes up to 10 minutes.

** Bookmark your page!!! I add my page to my bookmark toolbar for easy access

** Make sure to click "Push" from the dropdown menu when sending files to GitHub.

** Be sure that everything is spelled correctly!! That is something that we will be reminded of often when writing code.

 

 

I use GitHub a lot for a lot of projects. Most are private projects and cannot be shared. Please ask questions. Once setup correctly you should be good to go.
