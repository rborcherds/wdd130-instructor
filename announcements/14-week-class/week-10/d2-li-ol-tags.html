I am see quite a few students using the <ul><li> or <ol><li> list items. Personally I believe that this is overused and there are much better ways to write HTML.

Keep this in mind when deciding on if you should use them or not.

  • Are you creating a true list of items or just using the list tags to keep things straight in your mind?
  • Is there a specific step process, or thought process that must be followed in an exact order or not?

If you are not creating a proper list, the list item tags are not needed.

 

Let me show you what I mean by this.

In the Forms exercise you create a form that had these HTML elements

<form action="/my-handling-form-page" method="post">
  <ul>
    <li>
          <label for="name">Name:</label>
          <input type="text" id="name" name="user_name">
    </li>

      <li>
          <label for="mail">E-mail:</label>
          <input type="email" id="mail" name="user_email">
    </li>

      <li>
          <label for="msg">Message:</label>
          <textarea id="msg" name="user_message"></textarea>
        </li>

      <li class="button">
          <button type="submit">Send your message</button>
    </li>

    </ul>
  </form>

This same form can be written like this:

<form action="/my-handling-form-page" method="post">
  <div>  
       <label for="name">Name:</label>
      <input type="text" id="name" name="user_name">
  </div>
  <div>
      <label for="mail">E-mail:</label>
      <input type="email" id="mail" name="user_email">
  </div>
  <div>
       <label for="msg">Message:</label>
       <textarea id="msg" name="user_message"></textarea>
  </div>
  <button type="submit">Send your message</button>
</form>

Working copy is located at: https://borcherds-ralph.github.io/wdd130/examples/form-li.html

 

Another way to do this form is like this:

<form action="/my-handling-form-page" method="post">
  <fieldset>
     <legend>Contact info</legend>
     <label for="name">Name:</label>
     <input type="text" id="name" name="user_name">
  
     <label for="mail">E-mail:</label>
     <input type="email" id="mail" name="user_email">
  
     <label for="msg">Message:</label>
     <textarea id="msg" name="user_message"></textarea>
  
  </fieldset>
  <button type="submit">Send your message</button>
</form>

Working example: https://borcherds-ralph.github.io/wdd130/examples/form-2.html

 

One thing you will notice from these examples, I do not use the list elements at all. These examples are the way forms are created in the workplace.

 

Here is another example of something you do not have to use the list items for

<nav>
  <ul>
    <li><a href="/">Home</a></li>
    <li><a href="page1.html">Page 1</a></li>    <!- This must be changed to your page name -->
    <li><a href="page2.html">Page 2</a></li>    <!- This must be changed to your page name -->
  </ul>
</nav>

You can still do the navigation doing it this way by placing the grid definition on the <nav> and not the <ul>

<nav>
   <a href="/">Home</a></li>
   <a href="page1.html">Page 1</a></li> <!- This must be changed to your page name -->
   <a href="page2.html">Page 2</a></li> <!- This must be changed to your page name -->
</nav>

Write the CSS this way to still get the navigation layout. You have 2 less elements to deal with in doing the navigation for each item.

nav  {
    display: grid;
    grid-template-columns: repeat(auto-fit, minmax(6em, 1fr));          
}
nav a {....}
nav a:hover {.....}
                

Working example: https://borcherds-ralph.github.io/wdd130/examples/navigation.html
