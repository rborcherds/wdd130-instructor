There are a few things I wish to correct and explain from the HTML of the Dry Oar website. Please use these in your personal site. Most of these have already been submitted to the course designer to fix for future classes.

 

You will find developers using things incorrectly all the time. You need to know where to find the correct answers.

 

All of these come from https://developer.mozilla.org/en-US/docs/Web/HTML site. Expand the Reference -> HTML Elements area and you will see all the HTML tags. There are tags with a red trash icon next to them. Those are tags that have been removed from the specification and you cannot use them.  When in question about a tag name, use this list as a resource.

 

 

<br> Tag

Never use <br> to separate paragraphs. The <br> is only to be used when doing something like and address or make new lines in displaying a poem. REad this article for more info: https://developer.mozilla.org/en-US/docs/Web/HTML/Element/br

 

 

Heading Tags (h1, h2, h3, h4, h5, h6)

https://developer.mozilla.org/en-US/docs/Web/HTML/Element/Heading_Elements

Usage notes Heading information can be used by user agents to construct a table of contents for a document automatically. 
Avoid using heading elements to resize text. Instead, use the CSS font-size property. (Do not wrap <a href> tags inside of heading tags.)
Avoid skipping heading levels: always start from <h1>, followed by <h2>and so on. 
Use only one <h1>per page or view. It should concisely describe the overall purpose of the content.

If you do wrap an <a href> tag with a heading tag I will have you remove it. You must use standard CSS styling. I would recommend putting a class name on the link to style it. The course material did not follow this rule.

<section> Tag

https://developer.mozilla.org/en-US/docs/Web/HTML/Element/section

Use this if you have text that can stand on its own, it is related to the content of the page and has a heading tag. Use h2 to h6 tags in this section only. If you do not have a heading tag, then you should use some other tag such as <article>, <aside> or <div>.

If you are only using the element as a styling wrapper, use a <div>. An unwritten rule is that a <section> should logically appear in the outline of a document .

 

 

<article> tag

https://developer.mozilla.org/en-US/docs/Web/HTML/Element/article

A few of you are planning on creating a BLOG page. This is the perfect place to use the <article> tag.

A given document can have multiple articles in it; for example, on a blog that shows the text of each article one after another as the reader scrolls, each post would be contained in an <article> element, possibly with one or more <section> s within.

 

<address> tag

https://developer.mozilla.org/en-US/docs/Web/HTML/Element/address

Read this article carefully and look at the examples. This is one place you can use the <br> tag correctly.

 

 

<nav> tag

https://developer.mozilla.org/en-US/docs/Web/HTML/Element/nav

It's not necessary for all links to be contained in a <nav> element. <nav> is intended only for major block of navigation links; typically the <footer> element often has a list of links that don't need to be in a <nav> element .

 

<main> tag

https://developer.mozilla.org/en-US/docs/Web/HTML/Element/main

The content of a <main> element should be unique to the document. Content that is repeated across a set of documents or document sections such as sidebars, navigation links, copyright information, site logos, and search forms shouldn't be included unless the search form is the main function of the page.

 

<header> tag

https://developer.mozilla.org/en-US/docs/Web/HTML/Element/header

The <header> element is not sectioning content and therefore does not introduce a new section in the outline.

In this class, this tag holds the logo and navigation bar at the top of the page. You could use it in other places to denote the <header> of a section or article. For ARIA specification, which you will learn on your own, the <header> is a page banner section of text to help a screen reader know that this is at the top of that section/page.

 

 

<footer> tag

https://developer.mozilla.org/en-US/docs/Web/HTML/Element/footer

The <footer> element represents a footer for its nearest ancestor sectioning content or sectioning root element. The footer is just that. The end of the section, article, page.

One of the best ways to format a footer is to use a color that is not used on the page to denote the end of the page. Also make sure you have good color contrast and the text in the footer is easy to read.

This article does a great job explaining a page footer and ways to use it well:   https://howtogetonline.com/best-practices-for-designing-a-website-footer.php

 

<div> tag

https://developer.mozilla.org/en-US/docs/Web/HTML/Element/div

As a "pure" container, the <div> element does not inherently represent anything. Instead, it's used to group content so it can be easily styled using the class or id attributes, marking a section of a document as being written in a different language (using the lang attribute), and so on.
