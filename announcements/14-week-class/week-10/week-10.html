HTML FORMS

You will also learn about HTML forms. This exercise is a very basic form with a response. Pay close attention to the details of how the form is created and read the instructions carefully. Read each step first then do the what it says. Step 2 in Canvas tells you to add some JavaScript code to the page below the FORM element. The FORM element starts with <form .....> and ends with </form>. You add the code below the closing tag.

There are 3 ways to create labels for your forms. The exercise teaches the implicit way. You need to get used to doing them the explicit way and this is the way I want you to create them in your project.

This is the implicit way 

<label>Name <input type="text" name="name">
    </label>

This is the explicit way and the method I want you to use. 

<label for="name">Name</label>
    <input type="text" id="name" name="name">

The for= in the label has to match the id= name in the input. Remember in week 2 and 3 you learned that the ID has to be unique in an HTML page. This is one reason why.

The explicit method for doing the labels works the best for accessibility readers (For example: visually impaired users / screen readers for blind people ).



The third way is to do placehoder for the label which is not recommended for readability . You can use this to give example of the format for an input such as Phone Number, Zip/Postal Code format, number format, etc. with either of the above options as well. Just add the placeholder property. 

<input type="text" id="name" name="name" placeholder="Name">



A couple of other things you can do to your forms that make a difference. Use the required option to make the field required for submission and the first field you want to have typed in use the autofocus property. Here is an example where this will be where the cursor starts on the form and it is required. You will learn a lot more about these properties in WDD-230. 

<input type="text" name="myInput" autofocus required>
