3.  Absolute vs relative URLs for the images . When you start a src= with a /, you are telling the browser to look for the file starting at the homepage level of the site. If you do not start with the /, then you are telling the browser to find the file based upon the current folder the of the file location.

This is an absolute location:    <img class="logo" src=" /wwr/images/logo.png " alt="Dry Oar Logo">. This cannot find the file on GitHub but can on the local Live Server. To find this file on GitHub the correct code is something like this: <img class="logo" src="/wdd130/wwr/images/logo.png" alt="Dry Oar Logo">. This will not work on your local live server for testing. This is why a relative URL is best like this one: <img class="logo" src="images/logo.png" alt="Dry Oar Logo">. As long as the images folder is in the same folder as the index.html, the server can fint the file, no matter if it is on the local machine or on the server.

 

4.  Images . Quite a few of you have used images on the internet. If you are using one of the free image sites, you must download the image and put it into your images folder. The reason for this is that the image you are linking to has a watermark in it to show that it is copyrighted and thus illegal to use this way. You have to create an account on that site, download a copy of the file and put it onto your own server.

Do not use your own images if they are not about river running. The images must make sense for what the content is about. You will get more instruction on this in Week 8 when you have to gather the images and write the content for your personal site. Images will most likely have to be in your repo for your personal site, so I recommend practicing now with them.

 

5.  Background colors:   Quite a few of you are using white for the body background and the a few other places. Take a loo at the discussion forum as I have an image that has been marked with the main HTML tags as to where they will end up in week 7 when you do the layout. This week you will add a couple more HTML tags and this image shows those tags and where they go.

Putting a background color on paragraphs and headers is OK as long as there is a reason for it. In this design, there is no reason. Most of the time all the text in a section or div will have the same background color. The issue with doing the background color on just a paragraph or heading tag is that unless you add padding, the background color is just as tall as the text. Background colors tend to be much larger and you have multiple tags withing the background color. The exception is buttons. Make sure you do not use white for button background colors unless you have a reason. Buttons must stand out and call attention to them. I will have examples of buttons in week 10.

 

6.  Footer background color :  The standard on the internet is to make the <footer> background denote the end of the page. Make it a color that is not defined in your Site Plan. #3e3e3e is a very popular shade for footer. I will give examples of footer in week 9/10 in the discussion forum as that becomes a topic you have to address in your personal site.

 

7.  Inline Styling . This is the CSS code put into the HTML tags directly. DO NOT DO IT . This goes against the HTML5 standards of separating the markup with the styling. All styling must go into the CSS file. No image width or height allowed in the HTML.

Here is a common example with the bolded items that is not allowed:

<img src="https://borcherds-ralph.github.io/wdd130.." alt="image description" width="600px"  style="widht: 600px;" >
