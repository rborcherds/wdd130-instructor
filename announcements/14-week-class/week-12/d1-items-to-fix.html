I have seen these issues on multiple submissions and they must be fixed before the submission of the final.

  1. You must use the same CSS file for all your pages.
  2. You must set a max-width. If you do not then a lot of your layouts will not look good for grading.
  3. You must use the @media query like what was given in Week 7 of the Dry Oar website. This will allow you to display your site on a smartphone. There is another announcement specific to that.
  4. You must optimize images. I am seeing a lot of images over 1 MB in size. These have a tendency to slow down the loading of pages and the ability to scroll up and down.
    W03 Assignment: HTML for the Home Page - Step 8 has the information about optimizing the images. There is a YouTube video as well about it. More info about this below.
  5. Do not use % for placement unless you know exactly what to expect. Most new developers do not understand how this works correctly. The % is based upon the parent elements size. So if your parent element is 1600px wide then it is a % of that width. If you screen is smaller than that such as 1368px wide (common 15 in monitor") then it is a % of 1386px.
  6. Do not use position: absolute; This is creating issues with some layouts as I use a monitor much larger than what students use for grading. Very few website use position: absolute; it is useful in very few circumstances, such as creating a sticky header on your page or placing social media icons in a spot of your screen that never move as you scroll. Not good for placing text over an image.
  7. You must use the fonts and colors you chose in your site plan.
  8. The layouts must match what is in your wireframe in your site plan. I did post an image of my screen in the Week 11 Discussion Forum showing the 2 browsers open next to each other on 1 monitor.
  9. You must validate all submissions before you submit. Quite a few of you have validation issues that are easy to fix if you just validate.

 

Image Optimization helps

Here are a couple of articles with links to free image optimization tools

https://www.wpwebify.com/blog/image-optimization-image-optimizer-tools/

https://enviragallery.com/best-free-image-optimization-tools-for-image-compression/

 

Most images just need to have the size and PPI (Pixels Per Inch) changed. Ideally the best PPI for web display is 96 ppi. This is 95% of all monitors. Some very high-end displays like 150 ppi.  Think of it this way.

  • Newspapers print at 50 ppi
  • monitors default are 72 to 96ppi
  • Photographs on photo paper is 300 to 600 ppi
  • Magazines are 150 to 300 PPI
  • Super glossy magazine print are 800-1200 ppi. These images are very large in size on your hard drive and people will not wait for them to load on the screen.

Ideally you want your images to be no wider than 1600 px and if you have a max-size only make the banner images that wide. 

Logo should have a physical size of no more than 200 px wide.

Here is a better version for use on the site. This is 550 px by 250 px still at 300 ppi. This file size is 18.1 KB using the file type of webp. The file type does make a big difference.

 

This is the same size but at 100ppi. You can see a slight difference but when you make this smaller to fin the logo area you will not be able to tell. This file size is 18.1 kb.

 

This is an SVG version of the logo. This is the best option for logos only, not photographs This image can be resized and it will not get distorted.

I do not know the name of the fonts used so I changed it a little to a font a I have. This file size is 11.6 kb.

 

These are just some example so you can see that there are a lot of options to use for image optimizations  Same principles apply to photograph images.  PNG is the largest, JPG/JPEG is the second largest, WebP is a new format and is quite small.

There is even a newer format but it is not widely supported yet, it is called HEIF — High Efficiency Image File Format. This will get adopted in the future as it was only released. This was created by Apple so only Apple devices support this format at this time.

This is a good article that explains all the various formats and explains the pros and cons of each format.  https://kinsta.com/blog/image-file-types/#7--heif--high-efficiency-image-file-format

 

For this class only use the following:

  • JPG/JPEG/JFIF/PHPEG/PIP - All these extensions are for the JPEG (Joint Photographic Expert Group image) format. Highly recommend changing the extension to .jpg for all these image types.
  • PNG - Portable Network Graphics. 
  • WebP - Web Picture format - Created by Google
  • SVG - Scalable Vector Graphics

You will learn a lot more about graphics as you do more development and work with images.

 

Free software to help with web content creation

I personally use GIMP (https://gimp.org) for image manipulation and I use Inkscape (https://inkscape.org) for SVG images. Both are free as they are opensource software. If you go to install these on a Windows 10 or 11 machine you will get a security warning because you did not get the software from the Microsoft Store.  The software is safe to use.

There is another program that is a lot more complicated to use as it is for people that love to paint and draw, it is called Krita (https://krita.org). I highly recommend this software if you have a drawing tablet connected to your computer and you like to paint and draw. 

 

Here is a list of free software that can be used instead of a lot of very pricey software. I use all of these software programs.

  • Gimp (https://gimp.org) is similar to Adobe Photoshop
  • Inkscape (https://inkscape.org) is similar to Adobe Illustrator
  • Krita (https://krita.org) is a sketching and painting program designed for digital artists.
