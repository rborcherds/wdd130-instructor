Now for what to expect this week.

You will be learning how to make code comments. Commenting your code is a great way to remind yourself and other developers as to what is going on with the code and why you did certain things.

You will be learning about HTML References. This is how you access various things on the web and what the differences are for the various ways to write them. References are things like URL's, linking to specific parts of a page and how to find files on the server based upon where the file you are in is located. You will be using these all the time.

Next you will be learning about CSS selectors. Keep this in mind. What you learn about selectors in this class is just scratching the surface as to what you are able to do with selectors. Selectors is how you apply CSS to the various parts of your HTML. There are 4 basic types, Universal, Type, Class and ID. 

There is an aspect of the material that you will be taught on this topic that is not really used in the workplace. That is using ID names to apply CSS from a CSS file. ID names are generally used with JavaScript and not for normal CSS styling. You do have to know about it as you will see very few people doing this or JavaScript will be applying CSS to an element on a web page using the ID name. There will be some more information posted in the Random Help channel on this topic during the week.

As a professional developer, the Type and Class are used all the time along with Combinators and separators, Selectors (pseudo-class), Type Selectors. ID is seldom used due to the limiting factor of the ID name.

 

The next thing you will learn about is a CSS property called FLOAT. I personally have not used this in a web page since about 2014. It is not something you should use all the time but for very specific reason. Most developers do not use it quite correctly and can make the page unstable if not done correctly. There are also times when this will not be allowed to be used depending on what type of layout you are using.

 

You will then learn about the CSS Position property. This is what allows you to place something in a spot on the page and it never moves, such as the navigation bar staying at the top of the browser screen so you can see it even while you are scrolling. Or the social media icons that hover over the on the left or right side of a web page all the time.

You will then work on some Home page enhancements using what you have learned. This is optional but I highly encourage you do to this. It will give you more practice and experience.
